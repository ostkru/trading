Обзор API

API ПорталДанных.РФ предоставляет полный набор методов для управления продуктами, офферами, заказами и складами.

Базовый URL: https://api.portaldata.ru/v1

Пример полного URL: https://api.portaldata.ru/v1/products

Документация: https://api.portaldata.ru/docs/

Авторизация: Все запросы требуют API ключ в заголовке Authorization

Авторизация

GET /api/v1/products

Все запросы к API требуют авторизации. Используйте заголовок:

Authorization: Bearer <ваш_api_ключ>

Пример полного URL: https://api.portaldata.ru/v1/api/v1/products

Продукты

Создание продукта

POST /api/v1/products
Параметр Тип Обязательный Описание
name string Да Название продукта
vendor_article string Да Артикул поставщика
recommend_price number Да Рекомендуемая цена
brand string Да Бренд продукта
category string Да Категория продукта
description string Нет Описание продукта

Получение списка продуктов

GET /api/v1/products

Параметры запроса:

Параметр Тип Обязательный Описание
page integer Нет Номер страницы (по умолчанию: 1)
limit integer Нет Количество элементов на странице (по умолчанию: 20)
brand string Нет Фильтр по бренду
category string Нет Фильтр по категории

Офферы

Создание оффера

POST /api/v1/offers
Параметр Тип Обязательный Описание
product_id integer Да ID продукта
offer_type string Да Тип оффера: "sale", "sell", "buy", "purchase"
price_per_unit number Да Цена за единицу
available_lots integer Да Доступное количество лотов
tax_nds integer Да НДС в процентах
units_per_lot integer Да Количество единиц в лоте
warehouse_id integer Да ID склада
is_public boolean Да Публичный оффер (по умолчанию: true)
max_shipping_days integer Нет Максимальное количество дней доставки (по умолчанию: 0)

Заказы

Создание заказа

POST /api/v1/orders
Параметр Тип Обязательный Описание
offer_id integer Да ID оффера
quantity integer Да Количество лотов для заказа

Обновление статуса заказа

PUT /api/v1/orders/{id}/status
Параметр Тип Обязательный Описание
status string Да Новый статус: "pending", "confirmed", "processing", "shipped", "delivered", "cancelled", "rejected"

Склады

Создание склада

POST /api/v1/warehouses
Параметр Тип Обязательный Описание
name string Да Название склада
address string Да Адрес склада
latitude number Нет Широта координат
longitude number Нет Долгота координат

Статусы заказов

  • pending - Ожидает подтверждения (по умолчанию)
  • confirmed - Подтвержден продавцом
  • processing - В обработке
  • shipped - Отправлен
  • delivered - Доставлен
  • cancelled - Отменен
  • rejected - Отклонен

Лимиты API

  • Минутные лимиты: 60 запросов в минуту для всех методов
  • Дневные лимиты: 1000 запросов в день только для GET методов
  • HTTP 429: Возвращается при превышении лимитов
  • Лимиты применяются: К каждому API ключу отдельно
