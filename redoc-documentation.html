Полная документация по REST API для управления продуктами, офферами, заказами и складами. Авторизация по API-ключу обязательна для всех методов (заголовок Authorization).

Основные модули

  • Products - Управление продуктами (включая штрих-коды, статусы и медиаданные)
  • Offers - Управление предложениями (офферами)
  • Orders - Управление заказами
  • Warehouses - Управление складами

Система статусов продуктов

  • processing - В обработке (по умолчанию при создании)
  • Статус не может быть изменен через API
  • Классификация происходит внешним скриптом по полям category_id и brand_id
  • Офферы создаются только для продуктов с заполненными category_id и brand_id

Фильтры продуктов

  • owner - Фильтр по владельцу:
    • my - Продукты текущего пользователя
    • others - Продукты других пользователей
    • all - Все продукты (по умолчанию)
  • upload_status - Фильтр по статусу загрузки:
    • classified - Продукты с заполненными category_id и brand_id (по умолчанию)
    • processing - В обработке
    • not_classified - Продукты без category_id или brand_id

Система статусов заказов

  • pending - Ожидает подтверждения (по умолчанию)
  • confirmed - Подтвержден продавцом
  • processing - В обработке
  • shipped - Отправлен
  • delivered - Доставлен
  • cancelled - Отменен
  • rejected - Отклонен

Типы офферов

  • sale/sell - Оффер на продажу
  • buy/purchase - Оффер на покупку

Роли в заказах

  • initiator_user_id - Покупатель (создатель заказа)
  • counterparty_user_id - Продавец (владелец оффера)

Медиаданные продуктов

  • image_urls - Массив URL изображений (.jpg, .jpeg, .png, .gif, .webp)
  • video_urls - Массив URL видео файлов (.mp4, .avi, .mov, .wmv, .flv)
  • model_3d_urls - Массив URL 3D моделей (.obj, .fbx, .3ds, .dae, .stl)
  • Медиаданные сохраняются в отдельной таблице media
  • Поддерживается валидация URL и расширений файлов
  • Все операции с медиаданными выполняются в транзакциях

Новые возможности

  • Штрих-коды продуктов - Опциональное поле barcode для продуктов
  • Система статусов продуктов - Контроль классификации товаров
  • Географические фильтры - Фильтрация офферов по координатам
  • Пакетные операции - Массовое создание продуктов и офферов
  • Медиаданные - Поддержка изображений, видео и 3D моделей для продуктов

Создать продукт

Создаёт новый продукт с поддержкой медиаданных. Это метаданные, на основе которых строится вся схема ассортимента на проекте. Наивысший приоритет уделяется данным от производителя. Все поля обязательны, кроме description. Медиаданные (image_urls, video_urls, model_3d_urls) опциональны и сохраняются в отдельной таблице media.

ApiKeyAuth

required

vendor_article
required
string

recommend_price
required
number <float> > 0.01

Рекомендуемая цена за единицу товара (обязательное поле). Должна быть больше 0. Поддерживает десятичные значения (например: 15999.99, 100.50)

brand

string

category

barcode
string

Штрих-код продукта (опциональное поле)

video_urls
Array of strings <uri> [ items <uri > ]

Массив URL видео файлов продукта (опциональное поле). Поддерживаемые форматы: .mp4, .avi, .mov, .wmv, .flv

Массив URL 3D моделей продукта (опциональное поле). Поддерживаемые форматы: .obj, .fbx, .3ds, .dae, .stl

Возвращает список продуктов с пагинацией и медиаданными. Можно фильтровать по производителю: my (мои), others (чужие), all (все). Каждый продукт включает медиаданные (image_urls, video_urls, model_3d_urls).

Authorizations:
ApiKeyAuth
query Parameters

string
Default: "all"
Enum: "my" "others" "all"

Фильтр по владельцу продуктов: 'my' — только мои продукты, 'others' — только чужие продукты, 'all' — все продукты (по умолчанию)

upload_status
string
Default: "classified"
Enum: "pending" "not_classified" "classified"

Фильтр по статусу загрузки продуктов: 'classified' — продукты с заполненными category_id и brand_id (по умолчанию), 'processing' — в обработке, 'not_classified' — продукты без category_id или brand_id

page
integer >= 1
Default: 1

Номер страницы для пагинации (начиная с 1, по умолчанию 1)

limit
integer [ 1 .. 100 ]
Default: 10

Количество продуктов на страницу (от 1 до 100, по умолчанию 10)
